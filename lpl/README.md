# LPL Model Project Structure

This directory contains the LPL data collection, model, calibration, odds evaluation, and local dashboard tools.

## Daily Commands

```powershell
npm run lpl:health
npm run lpl:markets
npm run lpl:evaluate
npm run lpl:strict
npm run lpl:dashboard
```

Use `npm run lpl:health` first when the project feels messy or after a code/data change.

Clean temporary caches and archive old one-off notes/scripts:

```powershell
npm run lpl:cleanup:dry
npm run lpl:cleanup
```

## Main Directories

| Path | Purpose |
|---|---|
| `lpl/build-market-analysis.js` | Main market probability generator. |
| `lpl/odds-core.js` | Betting gate and EV/risk logic. This is the source of permission to bet. |
| `lpl/shared.js` | Shared CSV, parsing, math, team alias, and path helpers. |
| `lpl/config/` | Human-maintained model/display configuration. |
| `lpl/calibration/` | Calibration and model fitting scripts/coefs. |
| `lpl/backtest/` | Historical prediction, outcome joining, calibration, and recommendations. |
| `lpl/dashboard/` | Local Windows model dashboard and team insight export. |
| `lpl/odds-history/` | SQLite odds history, semi-auto collection, OP snapshots, and price-diff monitor. |
| `lpl/tools/` | Project maintenance tools such as health checks. |
| `lpl/archive/` | Archived one-off scripts/reports moved out of active paths. |
| `lpl/data/` | Current data and generated model outputs. |

## Data Layout

| `lpl/data/lpl_matches.csv` | Current merged LPL match list. |
| `lpl/data/lpl_map_details.csv` | Current merged map-level details. |
| `lpl/data/lpl_player_map_details.csv` | Current player-map details. |
| `lpl/data/history/all_matches.csv` | Historical match dataset for backtests/calibration. |
| `lpl/data/history/all_map_details.csv` | Historical map dataset for backtests/calibration. |
| `lpl/data/盘口分析/` | Generated reports, odds templates, evaluations, candidates, and dashboard exports. |

| `lpl/data/odds_history/odds.db` | Local SQLite odds snapshot database. |

## Important Guardrails

- Dashboard team ratings are display-layer analysis only.
- Betting decisions must still pass `lpl/odds-core.js`, `npm run lpl:evaluate`, `npm run lpl:strict`, and bankroll rules.
- The Dengfeng/Niepan cross-group display adjustment lives in `lpl/config/division-rating.json`.
- Do not treat `group_internal_score` as cross-group comparable. Use `rating_score` for display comparisons.
- Hard market bans for bad historical performance should stay in `lpl/odds-core.js`.

## Common Workflows

### Refresh Model Outputs

```powershell
npm run lpl:markets
npm run lpl:evaluate
npm run lpl:strict
npm run lpl:health
```

### Refresh Team Dashboard Data

```powershell
npm run lpl:dashboard:insights
npm run lpl:dashboard
```

### Backtest Current Model

```powershell
npm run lpl:backtest
npm run lpl:health
```

### Change Dengfeng/Niepan Display Rating

Edit:

```text
lpl/config/division-rating.json
```

Then run:

```powershell
npm run lpl:dashboard:insights
npm run lpl:health
```

### Clean Old Residue

Dry-run first:

```powershell
npm run lpl:cleanup:dry
```

Execute:

```powershell
npm run lpl:cleanup
```

The cleanup deletes temporary fetch/cache directories and archives one-off scripts/reports under:

```text
lpl/archive/cleanup-20260531/
```
