# LCK 盘口 EV 模型工具

这个副本已经改成 LCK 版本。核心目标不是预测胜负，而是用公开数据先判断比赛剧本，再评估盘口是否有正期望值。

```text
EV = 模型命中率 × 十进制赔率 - 1
```

只有 `EV > 0`、比赛剧本一致、盘口之间没有明显冲突时，才会进入 A/B 档。

## 一键使用

双击：

```text
一键更新LCK盘口数据.cmd
```

或在终端运行：

```powershell
npm run update
```

输出目录：

```text
lck\data\盘口分析
```

主要文件：

- `LCK盘口报告.md`：按比赛输出剧本、队伍状态、关键数据、EV 表格、推荐和风险提示。
- `赔率填写模板.csv`：把盘口赔率填到 `odds` 列。
- `赔率评估结果.csv`：计算隐含胜率、模型胜率、EV、风险等级和建议金额。
- `可下注候选.csv`：只保留 A/B 档且无明显冲突的候选。
- `不下注盘口说明.csv`：已填赔率但不下注的原因。
- `冲突检查.csv`：同场盘口逻辑冲突或需要确认剧本的提示。
- `队伍盘口命中率.csv`：LCK 队伍画像和强度/节奏指标。
- `英雄盘口模板.csv` / `英雄盘口评估结果.csv`：英雄使用盘口单独建模。

## 常用命令

```powershell
npm run collect       # 采集 LCK GOL 公开数据和小局详情
npm run collect:quick # 只采集赛程/队伍/选手/英雄汇总，不抓逐局页面
npm run markets       # 生成剧本、盘口概率、赔率模板和报告
npm run evaluate      # 填完赔率后只更新 EV 评估
npm run strict        # 严格筛选 A/B 候选
npm run heroes        # 生成和评估英雄使用盘口
```

默认赛事是：

```text
LCK 2026 Rounds 1-2
```

可以手动指定：

```powershell
node lck/collect-lck.js --tournament="LCK Cup 2026"
```

调试时只抓最近 N 场已结束比赛的小局：

```powershell
node lck/collect-lck.js --max-matches=5
```

## 模型逻辑

每场比赛先判断剧本，再看盘口：

- 强队速推碾压局：优先看强队让图、强队让击杀、时间小。
- 强队慢热终结局：优先看强队胜、弱队受让击杀、时间大。
- 弱队能咬住但难赢：优先看弱队 +1.5、弱队击杀受让、地图大 2.5。
- 混乱高击杀局：优先看总击杀大、时间大。
- 低击杀运营局：优先看总击杀小、时间大、强队胜。

风险等级：

- A：EV 明显为正，样本和剧本一致，无明显冲突。
- B：EV 为正但优势不厚或样本一般，小注。
- C：只观察，不下注。
- D：EV 为负或逻辑冲突，放弃。

资金规则默认以本金 `100` 为单位：单日总投入约 30 单位封顶，同场高度相关盘口最多 15 单位，高波动盘口如一血和英雄盘自动降权。

## 数据来源说明

当前 LCK 版本使用 Games of Legends 公开页面：

- 赛程和赛果
- 队伍汇总数据
- 选手数据
- 英雄数据
- 已结束比赛的小局页面、BP、蓝红方、时长、击杀、资源

GOL 小局页没有稳定提供首龙/首先锋逐局字段，所以模型用队伍汇总中的 `DRA%` 和 `HER%` 做资源控制替代。
